## 1. 安装与启动

### Windows
- 下载 `.msi` 安装包或绿色版 `.exe`
- 安装后启动即可
- 绿色版直接运行 `obsidian-attachments-voyager.exe`

### macOS
- 下载 `.dmg` 文件
- 拖拽 `.app` 到 Applications 文件夹
- 首次启动可能需要在「安全性与隐私」中允许运行

### Linux
- 下载 `.deb` 包（Debian/Ubuntu）或直接运行 ELF 二进制文件
- 需要 WebKitGTK 等依赖（参见 README）

### 使用前提
打开 Obsidian 的 **设置 → 文件与链接**，确认：
- 附件默认存放路径：**当前文件所在文件夹下指定的子文件夹**
- 子文件夹名称：**attachments**

> 本工具基于此规范识别附件的正确位置。

## 2. 首次扫描

1. 在顶部工具栏的输入框中输入仓库路径，或点击 **「选择」** 按钮选择文件夹
2. 勾选 **「缩略图」** 可生成图片预览（推荐开启，首次会稍慢）
3. 点击 **「扫描」** 按钮开始
4. 扫描分三个阶段，进度条会实时显示：
   - **收集文件**：遍历仓库目录
   - **解析 Markdown**：读取所有 .md 文件，提取图片引用
   - **生成缩略图**：为图片创建预览缩略图

> 后续扫描会使用增量模式，自动跳过未修改的文件，速度更快。

## 3. 理解扫描结果

扫描完成后，左侧侧边栏显示两个分类：

### 孤立附件（Orphan）
- 位于 `attachments` 目录中，但没有任何 Markdown 文件引用
- 通常是删除笔记后遗留的无用图片
- **修复方式**：删除文件

### 错位附件（Misplaced）
- 被 Markdown 文件引用，但不在正确的 `attachments` 目录下
- 例如：笔记在 `folder-A/note.md`，但引用的图片在 `folder-B/attachments/img.png`
- 正确位置应该是 `folder-A/attachments/img.png`
- **修复方式**：移动文件到正确目录

### 界面操作
- **画廊视图**：中间区域以缩略图卡片形式展示
- **三种显示模式**：缩略图 / 原图 / 无图
- **右侧详情面板**：点击卡片查看文件详情
- **搜索和筛选**：左侧可按文件名搜索、按类型过滤、按大小过滤

## 4. 修复问题

### 选择文件
| 操作 | 效果 |
|------|------|
| 左键单击 | 单选 / 取消选中 |
| Ctrl/Cmd + 单击 | 多选切换 |
| Shift + 单击 | 区间选择 |
| 「全选」按钮 | 选中当前分类下所有 |

### 执行修复
1. 选中需要修复的图片
2. 点击工具栏的 **「修复」** 按钮
3. 弹出确认对话框，显示：
   - ⚠️ **执行后无法恢复**
   - 建议提前备份重要图片
4. 点击 **「确认执行」** 开始修复

### 修复结果
- **孤立附件** → 直接从磁盘删除（不进回收站）
- **错位附件** → 移动到正确的 `attachments` 目录
- 如果目标位置已有同名文件，默认自动重命名（如 `image (1).png`）

> ⚠️ 修复操作不可撤销。建议在修复前使用备份功能。

## 5. 备份附件

选中图片后，工具栏会出现 **「备份 ▾」** 下拉按钮：

### 备份到目录
1. 点击「备份到目录」
2. 选择目标文件夹
3. 所有选中的图片会复制到该文件夹
4. 同名文件自动重命名

### 打包为 ZIP
1. 点击「打包为 ZIP」
2. 选择保存路径和文件名
3. 所有选中的图片打包为一个 ZIP 文件

> 备份操作不会修改原始文件，纯复制/打包。

## 6. 迁移与预处理

这一页现在包含两类目录整理工具：联动迁移，以及“汇总子目录 attachments”的预处理。

### 联动迁移

联动迁移用于将一篇笔记及其引用的所有附件搬到新位置：

1. 切换到 **「迁移与预处理」** 标签页
2. 选择要迁移的 Markdown 文件
3. 选择目标目录
4. 工具会自动：
   - 先移动笔记引用的所有附件到新目录的 `attachments/`
   - 再移动笔记文件本身
5. 冲突处理策略与修复相同

> 移动顺序为先附件后笔记，确保中途出错时笔记仍在原位。

### 预处理：汇总子目录 attachments

如果你让 AI 或手工重组过笔记目录，原来的 `attachments` 目录可能散落在很多子文件夹里。这个预处理就是先把它们集中起来：

1. 在 **「迁移与预处理」** 页的下半部分，选择你要整理的父级目录
2. 点击 **「预览汇总计划」**，先查看将要发生的操作
3. 工具会递归查找该目录下所有子文件夹中的 `attachments` 目录
4. 这些 `attachments` 目录里的文件会被统一移动到当前所选目录下的 `attachments/`
5. 原先那些已经搬空的 `attachments` 目录会被自动删除
6. 预处理完成后，回到扫描页重新扫描，再用现有“扫描 + 修复”流程把附件归位到各自正确的位置

> 这一步的价值在于先完成一次集中清理，避免以后还要手动回头检查哪些子文件夹里残留了空的 `attachments` 目录。

## 8. 设置与偏好

### 主题切换
标题栏右侧可选择四种主题：
- **跟随系统**：自动匹配操作系统的深色/浅色模式
- **亮色**
- **暗色**
- **羊皮纸**：温暖的复古风格

### 语言切换
标题栏右侧的语言按钮可切换中文/English。

### 缩略图缓存管理
- 底部状态栏有 **「清除缩略图缓存」** 按钮
- 清除后下次扫描会重新生成（一张图片生成 3 个尺寸的缩略图）
- 缩略图存储在仓库根目录的 `.voyager-gallery-cache/` 文件夹

### 数据存储
- 所有设置存储在 exe 同目录的 `voyager-data/` 文件夹
- 操作历史保存在 `voyager-data/ops-history.json`
- 绿色版卸载时删除整个文件夹即可

## 9. 常见问题

**Q: 修复操作可以撤销吗？**
A: 不可以。修复操作（删除/移动）一旦执行无法自动撤回。建议在修复前使用备份功能导出需要保留的图片。

**Q: 扫描速度慢怎么办？**
A:
- 关闭缩略图生成可大幅加快速度
- 第二次扫描会使用增量模式，自动跳过未修改的文件
- 缩略图只有首次生成慢，之后会从缓存读取

**Q: 为什么有些图片显示为「缩略图加载失败」？**
A: 可能是缩略图缓存损坏或格式不支持。尝试点击状态栏的「清除缩略图缓存」，然后重新扫描。

**Q: 数据存储在哪里？**
A: 用户设置和操作历史存储在 exe 同目录的 `voyager-data/` 文件夹中，不写入系统 AppData。缩略图缓存在仓库根目录的 `.voyager-gallery-cache/` 中。

**Q: 可以同时管理多个仓库吗？**
A: 可以。每次在工具栏输入不同的仓库路径即可切换。最近使用的 8 个仓库路径会自动记忆。

**Q: 为什么显示的缩略图数量是实际图片的 3 倍？**
A: 每张图片会生成 3 个不同尺寸的缩略图（64px / 256px / 1024px），用于不同的显示场景。缩略图总数 ÷ 3 = 实际图片数量。
